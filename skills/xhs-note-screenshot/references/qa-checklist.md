# 质量检查清单

在最终交付前依次执行这些检查。

## 结构检查

- `manifest.json` 存在，且其中的 `images` 数组非空。
- 第一张图使用模板 `xhs-note-with-author`，且恰好包含一个 `header.author` 块。
- 其他所有图使用模板 `xhs-note-body`，且**不含**任何 `header.author` 块。
- 每张图都恰好包含一个 `section.body`。
- 每张图都带有 `XHS Note Screenshot` 生成器 meta 标签。

## 头像 / 作者解析检查

- 用户通过参数传入的字段，无论档案内容如何，都使用 `cli` 来源。
- 用户没有传入、但档案提供的字段，使用 `profile` 来源。
- 用户和档案都没有提供的字段，使用 `placeholder` 来源。占位头像是一个圆形暖灰色 `#c9bfa6` 块，占位作者名为 `匿名`（zh）或 `Anonymous`（en）。
- 智能体永远不要凭空捏造未被提供的头像 URL 或作者名。

## 视觉检查

- 背景为柔和的牛皮纸色（`#f3e8d2`），与墨色文字对比度足够。
- 任何图上都不能有阴影、边框、渐变或装饰块。
- 头像渲染为圆形（`border-radius: 50%`），且未被拉伸。
- 作者名为粗体（字重 600），在缩略图尺寸下依然可读。
- 日期为暖灰色（`#7a6a52`），字号小于作者名，并紧贴在作者名下方。
- 正文字号至少 18px（CSS），行高至少 1.7。
- 段落不能被裁切，不能出现横向滚动条。
- 第一张图的内边距与正文图一致（CSS 40px）。

## Markdown 检查

- 输入文件中的 `**...**`、`*...*`、`` `...` ``、`# ...`、`- ...`、`1. ...`、`> ...`、`[...](...)` 已被解析为对应 HTML 标签，而不是被原样转义。
- 独占一行的 `---` 被识别为分页符号 —— 每条 `---` 之后的内容出现在新的 `note-NN.html` 文件中。
- 标题 `h1` / `h2` / `h3` 字号依次递减，且不会出现在第一张图以外的图上时挤压正文空间。
- `blockquote` 左侧有一条暖灰色竖线，文字为斜体。
- `code` / `pre code` 使用等宽字体，带浅墨色背景。
- `ul` / `ol` 缩进正确（28px），不与外层内边距重叠。

## 多图检查

- 如果输入能放进单张图，就不要生成第二张图。
- 如果输入被拆分，拆分点必须落在段落边界上，不能在句子中间。
- 即便正文被拆成很多页，作者块也绝不能在正文图上再次出现。

## 测试代码

`tests/avatar-display.test.mjs` 和 `tests/markdown-rendering.test.mjs` 是规范的检查工具。它们会针对 fixture 运行 `build-note.mjs`，检查生成的 HTML 文件，并对上述结构规则做断言。运行方式：

```bash
npm test              # 跑全部测试
npm run test:avatar   # 只跑头像 / 作者解析
npm run test:markdown # 只跑 Markdown 渲染
```
